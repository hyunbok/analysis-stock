## 1. 디자인 레퍼런스 분석

### 1.1 Coinone (coinone.co.kr) 분석

**컬러 시스템**
- 브랜드 강조색: `#0B59D5` (강한 파랑 계열)
- 배경: `#FFFFFF`, `#F8F8F9` (밝고 깔끔한 화이트)
- 주요 텍스트: `#17181B`, `#040505` (딥 차콜)
- 위험/매도 색상: `#E61919` (레드)
- 입력 필드 포커스: `border: 2px solid #0B59D5`
- 버튼 호버: `background: rgba(11,89,213,0.04)`

**레이아웃 특징**
- Flex 기반 반응형 레이아웃
- 데스크톱 최대 너비: 1440px
- 고정 헤더: 68px (데스크톱), 52px (모바일)
- 필터 버튼: height 32px, border-radius 6px
- 전환 애니메이션: 250ms

**폰트**
- Pretendard (한글 가독성 최적화)
- Noto Sans KR (폴백 폰트)
- 제목: 700 weight
- 본문: 400 weight, 14-16px

**UX 패턴**
- 코인 목록: 헤더 + 테이블 행 구조
- 검색 입력: 아이콘 통합, rounded border
- 배너: 블러 백드롭 효과
- 화이트 계열 라이트 테마 기본

### 1.2 Upbit (upbit.com) 분석

**컬러 시스템**
- 매수/상승: 밝은 녹색 (Green) — 한국 거래소 표준
- 매도/하락: 밝은 빨간색 (Red)
- 거래량 표시: 컬러 바 (강한 매수=밝은 녹색, 강한 매도=밝은 빨간색)
- 다크/라이트 테마 모두 지원

**차트 시스템**
- TradingView 기반 캔들스틱 차트
- 기술적 지표 오버레이 지원 (MA, RSI, MACD, Bollinger Bands)
- 차트 상단: 코인 정보 + 가격 + 등락률
- 우측: 거래량 막대

**레이아웃 특징 (데스크톱 트레이딩 화면)**
- 3단 구성: 차트(좌중앙) + 호가창(우측) + 주문창(우측 하단)
- 상단: 마켓 탭 바
- 좌측: 코인 목록 패널 (숨김/표시 토글)

**UX 패턴**
- TradingView 위젯 임베드
- 실시간 호가창: 가격 변동 시 행 색상 플래시
- 최우선 매수/매도 호가 강조 표시
- 거래 체결 시 거래량 막대 바 업데이트

### 1.3 글로벌 레퍼런스 (Binance, Bybit) 요약

**공통 패턴**
- 다크 모드 기본 (트레이딩 앱 표준)
- 바텀 네비게이션 (모바일): 홈, 마켓, 트레이딩, 자산, 더보기
- 트레이딩 화면: 풀스크린 차트 + 하단 탭(차트/호가창/주문)
- 실시간 가격 업데이트: 변동 방향에 따라 녹색/빨간색 플래시
- 주문창: 매수(Buy) 탭 / 매도(Sell) 탭 분리

## 2. 전체 디자인 방향

### 2.1 디자인 철학

**"Professional Clarity" — 전문성과 명확성의 균형**

CoinTrader는 개인 자동매매 트레이더를 위한 앱으로, 복잡한 데이터를 직관적으로 제공하는 것이 핵심이다. 초보 투자자도 어렵지 않게 접근할 수 있도록 명확한 시각적 계층(Visual Hierarchy)을 유지하면서, AI 자동매매라는 차별화된 기능을 전면에 부각시킨다.

**3대 원칙**

| 원칙 | 설명 |
|------|------|
| 데이터 우선 | 실시간 데이터가 중심. 광고, 불필요한 장식 최소화 |
| 액션 명확화 | 매수/매도, AI 활성화 등 핵심 액션은 눈에 즉시 띄게 |
| 다크 우선 | 장시간 화면 응시 시 눈 피로 최소화. 다크 모드 기본 |

### 2.2 Material 3 적용 방향

- **Dynamic Color** 비사용: 트레이딩 앱 특성상 일관된 컬러 팔레트 유지
- **Color Scheme**: 수동으로 정의한 시드 컬러(Seed Color) 기반 팔레트
- **Elevation**: Surface Tint 대신 명도 차이로 레이어 표현 (다크 테마에 적합)
- **Shape**: Medium (12dp radius) 기본 — 너무 둥글지 않게, 너무 각지지 않게

## 3. 컬러 시스템

### 3.1 브랜드 시드 컬러

```
Primary Seed: #1565C0 (코인원 파랑과 Material 3 사이의 균형)
```

### 3.2 핵심 의미 색상 (Semantic Colors)

트레이딩 앱에서 가장 중요한 색상은 매수/매도 색상이다. 한국 시장 표준을 따른다.

| 역할 | 색상 | Hex | 사용처 |
|------|------|-----|--------|
| 매수 / 상승 / 양봉 | 빨간색 | `#F23645` | 상승 가격, 매수 버튼, 양봉 캔들 |
| 매도 / 하락 / 음봉 | 파란색 | `#1E88E5` | 하락 가격, 매도 버튼, 음봉 캔들 |
| AI 활성화 | 민트/시안 | `#00BCD4` | AI ON 상태, AI 관련 배지 |
| 수익 (PnL+) | 연두/녹색 | `#26A69A` | 수익 표시, 성과 지표 양수 |
| 손실 (PnL-) | 주황/적색 | `#EF5350` | 손실 표시, 위험 경고 |
| 중립 | 그레이 | `#9E9E9E` | 보합, 변동 없음 |

> 참고: 한국 거래소(업비트, 코인원)는 상승=빨간색, 하락=파란색 컨벤션을 사용한다.
> 글로벌 거래소(바이낸스 등)는 상승=녹색, 하락=빨간색을 사용한다.
> CoinTrader는 한국 표준을 기본으로 하되, 설정에서 글로벌 스타일로 전환 가능하게 한다.

### 3.3 다크 테마 팔레트

```
Background (최하위):    #0A0A0F  (거의 검정에 가까운 네이비 블랙)
Surface (카드/패널):    #12131A  (짙은 네이비)
Surface Variant:        #1C1D26  (중간 네이비)
Surface Container:      #232434  (상단 앱바, 하단 네비 배경)
On Surface (주요 텍스트): #E8E9F0  (옅은 화이트)
On Surface Variant:     #9B9CB0  (보조 텍스트)
Outline:                #3A3B4E  (구분선, 테두리)
Primary:                #5C9BF5  (다크 테마용 파랑 — 라이트보다 밝게)
Primary Container:      #1565C0  (주요 버튼 배경)
```

### 3.4 라이트 테마 팔레트

```
Background:             #F5F6FA  (매우 옅은 블루그레이)
Surface:                #FFFFFF  (순백)
Surface Variant:        #F0F1F8  (카드 배경)
Surface Container:      #E8EAEF  (앱바, 네비 배경)
On Surface:             #1A1B23  (딥 차콜)
On Surface Variant:     #5C5E70  (보조 텍스트)
Outline:                #C8C9D8  (구분선)
Primary:                #1565C0  (메인 파랑)
Primary Container:      #DDEEFF  (버튼 배경 라이트)
```

### 3.5 컬러 사용 규칙

- 배경에 색상을 사용하지 않는다. 흰색/검정 계열 유지.
- 데이터 시각화 (차트 라인, 호가창 색상 바)에만 의미 색상 사용
- 브랜드 Primary 색상은 CTA 버튼, 탭 인디케이터, 링크에만 제한적으로 사용
- 매수/매도 버튼은 전체 너비로 배치하여 명확한 구분 제공

## 6. 네비게이션 구조

### 6.1 모바일 네비게이션 — Bottom Navigation Bar

Material 3의 `NavigationBar` 위젯을 사용한다.

```
┌─────────────────────────────────────────┐
│                                         │
│              [화면 컨텐츠]               │
│                                         │
├────────┬────────┬────────┬──────────────┤
│  홈    │ 트레딩 │  AI    │   더보기     │
│ (코인) │ (차트) │ (자동) │  (메뉴)      │
└────────┴────────┴────────┴──────────────┘
```

**탭 구성 (4개)**

| 탭 | 아이콘 | 화면 |
|----|--------|------|
| 홈 | `home` | 메인 코인 목록 |
| 트레이딩 | `candlestick_chart` | 트레이딩 화면 (코인 목록에서 진입) |
| AI 매매 | `smart_toy` | AI 자동매매 대시보드 |
| 더보기 | `menu` | 매매 내역, 설정, 거래소 설정 등 |

> 탭이 5개 이상이 되면 UX가 복잡해지므로 4개로 제한한다.
> "더보기" 탭은 Drawer 또는 리스트 메뉴로 서브 화면으로 이동

### 6.2 화면 전환 구조

```
스플래시
   └→ 로그인/회원가입 (미인증)
         └→ 메인 (인증 완료)
              │
    ┌─────────┼─────────┬──────────────┐
    ▼         ▼         ▼              ▼
 [홈 탭]  [트레이딩]  [AI 매매]   [더보기]
 코인목록   차트/호가창  대시보드    ├ 매매 내역
    │       주문창                  ├ 거래소 설정
    └→ 트레이딩 화면 (코인 탭)     └ 설정
         (push navigation)
```

### 6.3 앱바 (App Bar) 구성

**메인 화면**
```
┌─────────────────────────────────────────┐
│ CoinTrader    [알림 🔔]  [프로필 👤]   │
├─────────────────────────────────────────┤
│ [Upbit] [CoinOne]                       │  ← 거래소 탭
└─────────────────────────────────────────┘
```

**트레이딩 화면**
```
┌─────────────────────────────────────────┐
│ ← BTC/KRW  ₿  [별★]  [차트↗]  [...]  │
└─────────────────────────────────────────┘
```

## 7. 화면별 레이아웃 컨셉

### 7.1 스플래시 화면 (Splash Screen)

**목적**: 앱 초기화, 토큰 갱신, 세션 복구

```
┌─────────────────────────┐
│                         │
│                         │
│       [앱 로고 SVG]      │
│       CoinTrader        │
│    코인 자동매매 플랫폼   │
│                         │
│   [━━━━━━━━━━━━━━━━]     │  ← LinearProgressIndicator
│       로딩 중...         │
│                         │
│     v1.0.0              │
└─────────────────────────┘
```

**디자인 포인트**
- 배경: Primary 다크 그래디언트 (`#0D1B3E` → `#12131A`)
- 로고: 화이트 SVG (코인 + 차트 모티프)
- 로딩 바: LinearProgressIndicator (Material 3)
- 자동 전환: 토큰 유효 → 메인, 토큰 없음 → 로그인

### 7.2 로그인/회원가입 화면 (Auth Screen)

**목적**: 이메일 기반 인증

```
┌─────────────────────────┐
│  ←                      │
│                         │
│   [로고]                 │
│   CoinTrader에 오신      │
│   것을 환영합니다         │
│                         │
```

## 8. Light / Dark 테마 전략

### 8.1 기본 테마: 다크 모드 (Dark Mode First)

트레이딩 앱의 특성상 장시간 화면을 응시하는 사용자를 위해 다크 모드를 기본으로 한다.

```
이유:
- 장시간 트레이딩 시 눈 피로 감소
- 업비트, 바이낸스, 바이비트 등 글로벌 표준
- 차트 가독성 향상 (어두운 배경에서 캔들 더 잘 보임)
- OLED 화면 배터리 절약
```

### 8.2 테마 전환 전략

```dart
// Flutter ThemeMode 3가지 지원
enum AppThemeMode { light, dark, system }

// MaterialApp.themeMode 설정
themeMode: appThemeMode == AppThemeMode.system
    ? ThemeMode.system
    : appThemeMode == AppThemeMode.dark
        ? ThemeMode.dark
        : ThemeMode.light,
```

### 8.3 테마별 핵심 차이점

| 요소 | Light Mode | Dark Mode |
|------|------------|-----------|
| 앱 배경 | `#F5F6FA` | `#0A0A0F` |
| 카드/Surface | `#FFFFFF` | `#12131A` |
| 주요 텍스트 | `#1A1B23` | `#E8E9F0` |
| 보조 텍스트 | `#5C5E70` | `#9B9CB0` |
| 구분선 | `#C8C9D8` | `#3A3B4E` |
| 앱바 배경 | `#E8EAEF` | `#232434` |
| 네비바 배경 | `#E8EAEF` | `#232434` |
| 차트 배경 | `#FFFFFF` | `#0A0A0F` |
| 호가창 배경 | `#FAFAFA` | `#12131A` |

### 8.4 차트 테마 연동

TradingView WebView에 현재 앱 테마 정보를 JavaScript bridge로 전달하여 차트 테마를 동기화한다.

## 9. 반응형 전략 (모바일 → 데스크톱)

### 9.1 브레이크포인트 정의

```dart
// Responsive Breakpoints
class AppBreakpoints {
  static const double mobile = 480;     // ~480px: 모바일 세로
  static const double mobileL = 600;    // ~600px: 모바일 가로
  static const double tablet = 840;     // ~840px: 태블릿
  static const double desktop = 1200;   // ~1200px: 데스크톱
  static const double desktopL = 1440; // ~1440px+: 와이드
}
```

### 9.2 레이아웃 전환 전략

**모바일 (< 600px) — Phase 1 기본**
- Bottom Navigation Bar (4탭)
- 트레이딩: 탭 방식 (차트/호가창/주문 탭 전환)
- 코인 목록: 단일 컬럼 리스트
- 메뉴: "더보기" 탭 → 서브 화면

**태블릿 (600px ~ 1200px) — Phase 2**
- Navigation Rail (좌측 세로 네비)
- 트레이딩: 차트(좌) + 호가창/주문(우) 2단 분할
- 코인 목록: 2컬럼 그리드 또는 사이드 패널
- AI 대시보드: 2컬럼 카드 레이아웃

**데스크톱 (> 1200px) — Phase 2**
- Navigation Rail + Drawer 조합
- 트레이딩: 3단 레이아웃 (코인목록 | 차트 | 호가창+주문)
- 화면 최대 너비: 1440px 제한, 중앙 정렬

### 9.3 Flutter 반응형 구현 패턴

```dart
// LayoutBuilder 패턴
LayoutBuilder(
  builder: (context, constraints) {
    if (constraints.maxWidth < AppBreakpoints.tablet) {
      return MobileTradingView();  // 탭 방식
    } else {
      return DesktopTradingView(); // 2-3단 분할
    }
  },
)

// AdaptiveLayout (flutter_adaptive_scaffold)
AdaptiveLayout(
  primaryNavigation: SlotLayout(
    config: {
      Breakpoints.small: SlotLayout.from(
        key: Key('Bottom Nav'),
        builder: (_) => BottomNavigationBar(...),
      ),
      Breakpoints.medium: SlotLayout.from(
        key: Key('Nav Rail'),
        builder: (_) => NavigationRail(...),
      ),
    },
  ),
)
```

## 10. 트레이딩 UX 패턴

### 10.1 실시간 데이터 표시

**가격 플래시 애니메이션**
```
가격 변동 감지 → 배경색 플래시 (300ms) → 원래 배경색 복귀
- 상승: 배경 매수색(빨강) 옅게 플래시
- 하락: 배경 매도색(파랑) 옅게 플래시
- 텍스트 색상은 항상 방향성 유지
```

**숫자 애니메이션**
- 가격 업데이트 시: AnimatedSwitcher 또는 CountUp 애니메이션
- 너무 자주 업데이트되면 눈이 피로하므로 최소 200ms throttle 적용

**WebSocket 연결 상태 표시**
- 연결됨: 앱바 우측 소형 녹색 점 (●)
- 연결 중...: 황색 점 + 스피너
- 연결 끊김: 적색 점 + "재연결 중" 배너

### 10.2 호가창 (Order Book) UX

**시각화 원칙**
- 매도 호가: 상단 배치, 파랑색 계열
- 매수 호가: 하단 배치, 빨강색 계열
- 잔량 표시: 가격 뒤 컬러 바 (max 잔량 대비 상대 너비)
- 최우선 호가: 볼드 처리, 상대적으로 크게

**인터랙션**
- 호가 행 탭: 해당 가격을 주문창 가격 필드에 자동 입력
- 스크롤: 상하 스크롤로 더 많은 호가 표시
- 고정 현재가 구분선: 중간에 현재가 표시줄 고정

**업데이트 최적화**
- 개별 행 재빌드 최소화 (키 기반 업데이트)
- 변경된 행만 애니메이션: 변동 행 노랑 플래시 100ms
- 초당 최대 5회 업데이트 (throttle)

### 10.3 차트 인터랙션 (TradingView)

**시간봉 선택 (TimeFrame Chips)**
```
[1분] [5분] [15분] [30분] [1시] [4시] [일] [주]
```
- 선택된 칩: FilledTonalChip (Primary 색상)
- 미선택: OutlinedChip

**차트 기능 (TradingView 위젯)**
- 캔들스틱 기본
- 이동평균선 오버레이 (MA5, MA10, MA20, MA60, MA120)
- VWAP 라인
- 볼린저 밴드
- RSI 서브 패널
- MACD 서브 패널
- 거래량 막대 (하단)

**모바일 제스처**
- 핀치-줌: 차트 확대/축소
- 수평 드래그: 시간 이동
- 탭: 크로스헤어 + 캔들 정보 팝업

### 10.4 주문창 UX

**주문 유형 전환**
- SegmentedButton: [매수 | 매도]
- 전환 시 버튼 색상 전환 (매수: 빨강, 매도: 파랑)

**가격 입력 최적화**
- 숫자 키패드 자동 표시 (키보드 타입: `TextInputType.number`)
- 한국 거래소: 호가단위(1원, 5원, 10원...)에 맞춘 스텝 버튼 (↑ ↓)
- 호가창 탭 시 자동 입력

**수량 비율 버튼**
- [25%] [50%] [75%] [100%] 칩 → 보유 잔고 대비 비율
- 탭 시 수량 자동 계산

**주문 확인**
- 모바일: 주문 버튼 탭 → 바텀시트 확인 다이얼로그
- 오류: SnackBar로 인라인 피드백 (ex: "잔고 부족")

### 10.5 AI 매매 UX

**AI ON/OFF 전환**
- 대형 Switch 컴포넌트 (오해 없는 명확한 상태 표시)
- ON 전환 시: "AI 자동매매를 시작합니다. 최대 투자금: OOO원" 확인 다이얼로그
- OFF 전환 시: 즉시 적용, Snackbar 확인

**AI 상태 실시간 표시**
- 분석 중: 애니메이션 아이콘 + "장세 분석 중..." 텍스트
- 대기 중: "다음 분석까지 3:45" 카운트다운 타이머
- 매매 실행: "BTC 매수 주문 제출 중..." 진행 상태

**전략 표시**
- 현재 전략: ChipList 형태 (TrendMA 눌림목, VWAP 밴드 등)
- 장세 분류: 원형 프로그레스 + 퍼센트

## 11. 컴포넌트 설계 원칙

### 11.1 핵심 공통 컴포넌트 목록

| 컴포넌트명 | 설명 |
|-----------|------|
| `CoinListTile` | 코인 목록 행 (아이콘 + 이름 + 가격 + 등락률) |
| `PriceText` | 실시간 가격 텍스트 (플래시 애니메이션 포함) |
| `ChangeRateText` | 등락률 텍스트 (색상 + 화살표 자동 적용) |
| `OrderBookRow` | 호가창 행 (가격 + 잔량 + 컬러바) |
| `AiStatusBadge` | AI 상태 배지 (ON/OFF/분석중) |
| `MarketRegimeCard` | 장세 분류 카드 (추세/횡보/전환) |
| `PnlText` | 손익 텍스트 (양수=수익색, 음수=손실색) |
| `ExchangeChip` | 거래소 선택 칩 |
| `TradeHistoryTile` | 매매 내역 행 |
| `OrderButton` | 매수/매도 주문 버튼 (전체 너비) |

### 11.2 컴포넌트 상태 정의

모든 주요 컴포넌트는 다음 상태를 명확히 처리해야 한다:

- **Loading**: Shimmer Effect (Skeleton UI)
- **Empty**: 빈 상태 일러스트 + 안내 텍스트
- **Error**: 오류 아이콘 + 재시도 버튼
- **Loaded**: 정상 데이터 표시
- **Realtime**: WebSocket 연결 중 실시간 업데이트

### 11.3 접근성 (Accessibility)

- 모든 인터랙티브 요소: 최소 44×44dp 터치 영역
- 색상만으로 의미 전달 금지: 아이콘 또는 텍스트 병행
- 다크/라이트 모두 WCAG 4.5:1 이상 명도 대비
- Semantics 위젯으로 스크린 리더 지원
- 글자 크기 조절: 시스템 폰트 크기 설정 반영 (`textScaleFactor`)

## 12. Stitch 화면 생성 계획

디자인 시안은 Google Stitch를 활용하여 다음 순서로 생성한다.

### 12.1 생성 우선순위

| 순위 | 화면 | 디바이스 | 비고 |
|------|------|---------|------|
| 1 | 메인 (코인 목록) | MOBILE | 핵심 화면 |
| 2 | 트레이딩 (차트+호가창+주문) | MOBILE | 가장 복잡한 화면 |
| 3 | AI 자동매매 대시보드 | MOBILE | 차별화 기능 |
| 4 | 로그인/회원가입 | MOBILE | 첫 진입 화면 |
| 5 | 매매 내역 | MOBILE | 통계/리포트 |
| 6 | 거래소 설정 | MOBILE | 설정 화면 |
| 7 | 설정 | MOBILE | 설정 화면 |
| 8 | 스플래시 | MOBILE | 단순 화면 |

### 12.2 Stitch 프롬프트 가이드라인

각 화면 생성 시 다음 요소를 프롬프트에 포함한다:

```
- 앱명: CoinTrader (코인 트레이딩 앱)
- 테마: Material 3 Dark Mode
- 플랫폼: 모바일 (Flutter)
- 컬러: Primary #1565C0, 매수 #F23645 (빨강), 매도 #1E88E5 (파랑)
- 폰트: Inter (숫자), Pretendard (한글)
- 언어: 한국어
- 레퍼런스: 업비트, 코인원 스타일
```

### 12.3 변형(Variant) 생성 계획

핵심 화면(메인, 트레이딩)에 대해 다음 변형을 생성하여 비교한다:

- **COLOR_SCHEME 변형**: 다크/라이트 2가지
- **LAYOUT 변형**: 정보 배치 방식 탐색
- **창의성 범위**: EXPLORE (균형 탐색)
