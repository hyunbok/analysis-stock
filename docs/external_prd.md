## 1. 인증 관련

### 1.1 소셜 로그인 (Google, Apple)
- **디자인 컨셉**: 로그인 화면(7.2)에 "소셜 구분선(── 또는 ──)" 및 소셜 로그인 영역 존재
- **PRD 현황**: 이메일 기반 인증만 정의 (`POST /api/v1/auth/register, login`)
- **필요 작업**: OAuth2 Provider 추가 (Google, Apple Sign-In), API 엔드포인트, Flutter 소셜 로그인 패키지
- **우선순위 제안**: Phase 2 (핵심 기능 아님, UX 편의)

### 1.2 비밀번호 찾기/재설정
- **디자인 컨셉**: 로그인 화면(7.2) 하단에 "비밀번호 찾기" 링크
- **PRD 현황**: 비밀번호 관련 API 없음
- **필요 작업**: `POST /api/v1/auth/forgot-password`, `POST /api/v1/auth/reset-password`, 이메일 발송 서비스
- **우선순위 제안**: Phase 1 (필수 기능)

### 1.3 생체 인증 (Face ID / Fingerprint)
- **디자인 컨셉**: 설정 화면(7.8)에 "생체 인증 [OFF/ON]" 토글
- **PRD 현황**: 언급 없음
- **필요 작업**: Flutter `local_auth` 패키지, 생체 인증 후 JWT 발급 플로우
- **우선순위 제안**: Phase 2

## 2. UI/UX 기능

### 2.1 가격 색상 설정 (한국식/글로벌 전환)
- **디자인 컨셉**: 컬러 시스템(3.2)에 "설정에서 글로벌 스타일로 전환 가능", 설정 화면(7.8)에 "가격 색상 [한국식 ▼]"
- **PRD 현황**: 언급 없음. users 테이블에 해당 컬럼 없음
- **필요 작업**: users 테이블에 `price_color_style` 컬럼 추가 (korean/global), 클라이언트 테마 분기
- **우선순위 제안**: Phase 1 (글로벌 사용자 고려 시)

### 2.2 코인 검색 기능
- **디자인 컨셉**: 메인 화면(7.3)에 SearchBar ("🔍 코인 검색...")
- **PRD 현황**: `GET /api/v1/coins`만 정의, 검색 파라미터 미정의
- **필요 작업**: `GET /api/v1/coins?q={keyword}` 검색 파라미터 추가 또는 클라이언트 로컬 필터
- **우선순위 제안**: Phase 1 (기본 UX)

### 2.3 WebSocket 연결 상태 UI 표시
- **디자인 컨셉**: 트레이딩 UX(10.1)에 연결 상태 표시 (녹색=연결, 황색=연결중, 적색=끊김)
- **PRD 현황**: WebSocket 기술 스펙만 정의, UI 상태 표시 미정의
- **필요 작업**: 클라이언트 WebSocket 상태 관리 + AppBar 인디케이터
- **우선순위 제안**: Phase 1 (실시간 트레이딩 필수)

### 2.4 관심 코인 스와이프 추가
- **디자인 컨셉**: 메인 화면(7.3) "행 우측 스와이프 → 별 아이콘 버튼 노출"
- **PRD 현황**: watchlist CRUD API만 정의, 인터랙션 패턴 미정의
- **필요 작업**: Flutter Dismissible/Slidable 위젯 구현
- **우선순위 제안**: Phase 1 (UX 상세)

## 3. AI 자동매매 관련

### 3.1 AI 전체 ON/OFF 마스터 스위치
- **디자인 컨셉**: AI 대시보드(7.6)에 "전체 AI 매매 [ON/OFF]" 최상위 스위치
- **PRD 현황**: 코인별 `ai_trading_configs`만 정의, 전체 ON/OFF 없음
- **필요 작업**: `PATCH /api/v1/ai-trading/master-switch` API 또는 users 테이블에 `ai_trading_enabled` 컬럼
- **우선순위 제안**: Phase 1 (안전장치)

### 3.2 AI 분석 카운트다운 타이머
- **디자인 컨셉**: AI UX(10.5) "다음 분석까지 3:45" 카운트다운
- **PRD 현황**: Celery Beat 5분 주기만 정의, 다음 실행 시간 API 없음
- **필요 작업**: 마지막 분석 시간 기반 카운트다운 계산 (클라이언트) 또는 `GET /api/v1/ai-trading/next-run` API
- **우선순위 제안**: Phase 1 (대시보드 UX)

### 3.3 장세 분류 퍼센트 표시
- **디자인 컨셉**: AI 대시보드(7.6) "추세 75%, 횡보 20%, 전환 5%" 세그먼트
- **PRD 현황**: Trend/Range/Transition 분류만 정의, 확률/퍼센트 응답 미정의
- **필요 작업**: AI 판단 결과에 confidence score 포함, API 응답 스키마 확장
- **우선순위 제안**: Phase 1 (AI 대시보드 핵심 정보)

### 3.4 손익 미니 차트
- **디자인 컨셉**: AI 대시보드(7.6) "오늘의 손익" 카드 내 미니그래프
- **PRD 현황**: `GET /api/v1/ai-trading/stats/daily` 통계 API만 정의, 차트 데이터 포맷 미정의
- **필요 작업**: 시간대별 누적 손익 데이터 API 또는 daily_pnl_reports에 시간대별 스냅샷 추가
- **우선순위 제안**: Phase 2 (시각화 고도화)

## 4. 알림 관련

### 4.1 알림 설정 세분화
- **디자인 컨셉**: 설정 화면(7.8)에 "가격 알림", "AI 매매 알림", "체결 알림" 개별 토글
- **PRD 현황**: M12(Phase 2)에 "푸시 알림" 한 줄 언급만
- **필요 작업**: 알림 설정 테이블/컬럼, FCM 연동, 알림 유형별 발송 로직
- **우선순위 제안**: Phase 2 (PRD와 동일)

### 4.2 알림 화면 (알림 목록)
- **디자인 컨셉**: AppBar(6.3)에 알림 벨 아이콘 🔔 존재
- **PRD 현황**: 알림 목록 화면/API 미정의
- **필요 작업**: notifications 테이블, `GET /api/v1/notifications` API, 알림 목록 화면
- **우선순위 제안**: Phase 2

## 5. 기타

### 5.1 이용약관 / 개인정보처리방침
- **디자인 컨셉**: 설정 화면(7.8)에 "이용약관 / 개인정보처리" 링크
- **PRD 현황**: 언급 없음
- **필요 작업**: 약관 콘텐츠 작성, WebView 또는 마크다운 뷰어
- **우선순위 제안**: Phase 1 (법적 필수)

### 5.2 프로필 수정 화면
- **디자인 컨셉**: 설정 화면(7.8)에 "닉네임", "[프로필 수정]" 버튼
- **PRD 현황**: `GET,PUT /api/v1/auth/me` API 존재, 별도 프로필 화면 미정의
- **필요 작업**: 프로필 수정 화면 UI (닉네임, 이메일 변경 등)
- **우선순위 제안**: Phase 1 (API는 있으나 화면 미정의)

### 5.3 앱바 내 프로필 아이콘
- **디자인 컨셉**: 메인 AppBar(6.3)에 프로필 👤 아이콘
- **PRD 현황**: 프로필 이미지/아바타 관련 정의 없음 (users 테이블에 avatar 컬럼 없음)
- **필요 작업**: users 테이블에 `avatar_url` 컬럼 추가 또는 이니셜 아바타 생성
- **우선순위 제안**: Phase 2
